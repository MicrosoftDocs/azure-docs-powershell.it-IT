# Restart-AzServiceFabricManagedNodeType

## Sinossi
Riavviare nodi specifici dal tipo di nodo.

## SINTASSI

```
Restart-AzServiceFabricManagedNodeType [-ResourceGroupName] <String> [-ClusterName] <String> [-Name] <String>
 -NodeName <String[]> [-ForceRestart] [-PassThru] [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## Descrizione
Riavviare nodi specifici dal tipo di nodo. I nodi del tessuto del servizio verranno disabilitati prima di riavviare le VM e riabilitati quando torneranno. Se l'operazione viene eseguita nei tipi di nodi primari, potrebbe essere necessario un po' di tempo perché potrebbe non riavviare tutti i nodi contemporaneamente. Use-ForceRestart forza l'operazione anche se il tessuto di servizio non è in grado di disabilitare i nodi, ma può essere usato con cautela perché potrebbe causare perdite di dati se nel nodo sono in esecuzione carichi di lavoro con stato.

## ESEMPI

### Esempio 1
```powershell
$rgName = "testRG"
$clusterName = "testCluster"
$NodeTypeName = "nt1"
Restart-AzServiceFabricManagedNodeType -ResourceGroupName $rgName -ClusterName $clusterName  -Name $NodeTypeName -NodeName nt1_0, nt1_3
```

Riavviare il nodo 0 e 3 nel tipo di nodo.
