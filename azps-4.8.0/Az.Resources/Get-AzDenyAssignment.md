# Get-AzDenyAssignment

## Sinossi

Per impostazione predefinita, elenca tutte le assegnazioni di negazione nell'abbonamento Azure selezionato.
USA i rispettivi parametri per elencare le assegnazioni a un utente specifico oppure per elencare le assegnazioni in un gruppo di risorse o una risorsa specifico.

## Descrizione
Usare il comando Get-AzDenyAssignment per elencare tutte le assegnazioni di negazione valide per un ambito.
Senza parametri, questo comando restituisce tutte le assegnazioni di negazione effettuate sotto l'abbonamento.
Questo elenco può essere filtrato usando i parametri di filtro per Principal, nega il nome e l'ambito dell'assegnazione.
Per specificare un utente, USA SignInName o i parametri di Azure AD ObjectId.
Per specificare un gruppo di sicurezza, usare il parametro di Azure AD ObjectId.
Per specificare un'applicazione Azure AD, USA i parametri ServicePrincipalName o ObjectId.
L'ambito in cui viene negato l'accesso può essere specificato.
Il valore predefinito è l'abbonamento selezionato.
L'ambito dell'assegnazione Deny può essere specificato usando una delle combinazioni di parametri seguenti a.
Ambito: questo è l'ambito completo che inizia con/subscriptions/ \<subscriptionId\> .
In questo modo verranno filtrate le assegnazioni che sono effettive in quel particolare ambito, vale a dire tutte le attività negate in tale ambito e successive.
b.
ResourceGroupName-nome di un gruppo di risorse sotto l'abbonamento.
In questo modo verranno filtrate le assegnazioni effettive al gruppo di risorse specificato, ossia tutte le assegnazioni negate in tale ambito e successive.
c.
ResourceName, ResourceType, ResourceGroupName e (facoltativamente) ParentResource-identifica una determinata risorsa sotto l'abbonamento e filtra le assegnazioni valide per l'ambito della risorsa.
Per determinare l'accesso negato per un determinato utente nell'abbonamento, usare l'opzione ExpandPrincipalGroups.
Verranno elencate tutte le assegnazioni di negazione assegnate all'utente e i gruppi a cui l'utente fa parte.
