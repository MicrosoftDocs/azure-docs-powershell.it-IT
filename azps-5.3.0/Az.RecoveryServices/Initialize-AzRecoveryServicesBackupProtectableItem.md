# Initialize-AzRecoveryServicesBackupProtectableItem

## Sinossi
Questo comando attiva l'individuazione di eventuali elementi non protetti del tipo di carico di lavoro specifico nel contenitore indicato. Se l'applicazione DB non Ã¨ protetta automaticamente, usare questo comando per scoprire le nuove DBs ogni volta che vengono aggiunte e procedere alla loro protezione.

## SINTASSI

```
Initialize-AzRecoveryServicesBackupProtectableItem [-Container] <ContainerBase> [-WorkloadType] <WorkloadType>
 [-PassThru] [-VaultId <String>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## Descrizione
il cmdlet richiede carichi di lavoro specifici all'interno di un contenitore. In questo modo viene attivata un'operazione che crea elementi protettivi.

## ESEMPI

### Esempio 1
```powershell
PS C:\> Initialize-AzRecoveryServicesProtectableItem -Container $Container -WorkloadType "MSSQL"
```

Il cmdlet esegue un'operazione di individuazione per nuovi elementi protettivi.
