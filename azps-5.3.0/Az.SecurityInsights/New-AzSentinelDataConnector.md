## Descrizione
Il cmdlet **New-AzSentinelAlertRule** crea una regola analitica (avviso) nell'area di lavoro specificata.
Ãˆ necessario specificare uno dei parametri, ad esempio-AzureActiveDirectory, per specificare il tipo di regola di avviso da creare.  Ogni tipo ha diversi parametri obbligatori.
Puoi usare il parametro *Confirm* e $ConfirmPreference variabile di Windows PowerShell per controllare se il cmdlet richiede conferma.
Nota: non tutti i connettori dati disponibili nel portale sono avaialble tramite API.

## ESEMPI

### Esempio 1
```powershell
PS C:\> $DataConnector = New-AzSentinelDataConnector -ResourceGroupName "MyResourceGroup" -WorkspaceName "MyWorkspaceName" -AzureSecurityCenter -Alerts Enabled -SubscriptionId ((Get-AzContext).Subscription.Id)
```

Questo esempio crea un **DataConnector** per Azure Security Center nell'area di lavoro specificata e lo archivia nella variabile $DataConnector.

### Esempio 2
```powershell
PS C:\> $DataConnector = New-AzSentinelDataConnector -ResourceGroupName "MyResourceGroup" -WorkspaceName "MyWorkspaceName" -MicrosoftCloudAppSecurity -Alerts Enabled -DiscoveryLogs Disabled
```

Questo esempio crea un **DataConnector** per la sicurezza dell'app cloud Microsoft nell'area di lavoro specificata e lo archivia nella variabile $DataConnector.
