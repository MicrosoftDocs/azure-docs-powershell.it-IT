# Update-AzVMWarePrivateCloud

## Sinossi
Aggiornare un cloud privato

## SINTASSI

### UpdateExpanded (impostazione predefinita)
```
Update-AzVMWarePrivateCloud -Name <String> -ResourceGroupName <String> [-SubscriptionId <String>]
 [-IdentitySource <IIdentitySource[]>] [-Internet <InternetEnum>] [-ManagementClusterSize <Int32>]
 [-Tag <Hashtable>] [-DefaultProfile <PSObject>] [-AsJob] [-NoWait] [-Confirm] [-WhatIf] [<CommonParameters>]
```

### UpdateViaIdentityExpanded
```
Update-AzVMWarePrivateCloud -InputObject <IVMWareIdentity> [-IdentitySource <IIdentitySource[]>]
 [-Internet <InternetEnum>] [-ManagementClusterSize <Int32>] [-Tag <Hashtable>] [-DefaultProfile <PSObject>]
 [-AsJob] [-NoWait] [-Confirm] [-WhatIf] [<CommonParameters>]
```

## Descrizione
Aggiornare un cloud privato

## ESEMPI

### Esempio 1: aggiornare le dimensioni del cloud privato per nome
```powershell
PS C:\> Update-AzVMWarePrivateCloud -Name azps-test-cloud -ResourceGroupName azps-test-group -ManagementClusterSize 4

Location      Name            Type
--------      ----            ----
australiaeast azps-test-cloud Microsoft.AVS/privateClouds
```

Aggiornare le dimensioni del cloud privato per nome

### Esempio 2: aggiornare le dimensioni dell'oggetto cloud privato per input
```powershell
PS C:\> Get-AzVMWarePrivateCloud -ResourceGroupName azps-test-group -Name azps-test-cloud | Update-AzVMWarePrivateCloud -ManagementClusterSize 4

Location      Name            Type
--------      ----            ----
australiaeast azps-test-cloud Microsoft.AVS/privateClouds
```

Aggiornare le dimensioni dell'oggetto cloud privato per input

## Note

ALIAS

PROPRIETÀ COMPLESSE DI PARAMETRI

Per creare i parametri descritti di seguito, Costruisci una tabella hash contenente le proprietà appropriate. Per informazioni sulle tabelle hash, eseguire Get-Help about_Hash_Tables.


IDENTITYSOURCE <IIdentitySource [] >: Single Sign-on per le origini delle identità
  - `[Alias <String>]`: Nome NetBIOS del dominio
  - `[BaseGroupDn <String>]`: Nome distinto di base per i gruppi
  - `[BaseUserDn <String>]`: Nome distinto di base per gli utenti
  - `[Domain <String>]`: Nome DNS del dominio
  - `[Name <String>]`: Nome dell'origine di identità
  - `[Password <String>]`: La password dell'utente di Active Directory con un minimo di accesso di sola lettura a base DN per utenti e gruppi.
  - `[PrimaryServer <String>]`: URL del server primario
  - `[SecondaryServer <String>]`: URL del server secondario
  - `[Ssl <SslEnum?>]`: Proteggere le comunicazioni LDAP con il certificato SSL (LDAPs)
  - `[Username <String>]`: ID di un utente di Active Directory con un minimo di accesso di sola lettura a base DN per utenti e gruppi

INPUTOBJECT <IVMWareIdentity> : parametro Identity
  - `[AuthorizationName <String>]`: Nome dell'autorizzazione del circuito ExpressRoute nel cloud privato
  - `[ClusterName <String>]`: Nome del cluster nel cloud privato
  - `[HcxEnterpriseSiteName <String>]`: Nome del sito aziendale di HCX nel cloud privato
  - `[Id <String>]`: Percorso identità risorse
  - `[Location <String>]`: Area di Azure
  - `[PrivateCloudName <String>]`: Nome del cloud privato
  - `[ResourceGroupName <String>]`: Nome del gruppo di risorse. Il nome è senza distinzione tra maiuscole e minuscole.
  - `[SubscriptionId <String>]`: ID dell'abbonamento di destinazione.
