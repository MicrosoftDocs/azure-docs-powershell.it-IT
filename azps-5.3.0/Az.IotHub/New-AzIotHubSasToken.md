# New-AzIotHubSasToken

## Sinossi
Genera un token SAS per un hub, un dispositivo o un modulo di destinazione.

## SINTASSI

### ResourceSet (impostazione predefinita)
```
New-AzIotHubSasToken [-ResourceGroupName] <String> [-IotHubName] <String> [-DeviceId <String>]
 [-ModuleId <String>] [-KeyName <String>] [-KeyType <PSKeyType>] [-Duration <Int32>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObjectSet
```
New-AzIotHubSasToken [-InputObject] <PSIotHub> [-DeviceId <String>] [-ModuleId <String>] [-KeyName <String>]
 [-KeyType <PSKeyType>] [-Duration <Int32>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### ResourceIdSet
```
New-AzIotHubSasToken [-ResourceId] <String> [-DeviceId <String>] [-ModuleId <String>] [-KeyName <String>]
 [-KeyType <PSKeyType>] [-Duration <Int32>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## Descrizione
Per i token SAS per i dispositivi, il parametro policy viene usato per accedere solo al registro di sistema del dispositivo. Di conseguenza, il criterio dovrebbe avere accesso in lettura al registro di sistema.
Per i token Hub molto il criterio fa parte della SAS.

## ESEMPI

### Esempio 1
```powershell
PS C:\> New-AzIotHubSasToken -ResourceGroupName "myresourcegroup" -IotHubName "myiothub"
```

Genera un token SAS Hub molto con il criterio iothubowner e la chiave primaria.

### Esempio 2
```powershell
PS C:\> New-AzIotHubSasToken -ResourceGroupName "myresourcegroup" -IotHubName "myiothub" -KeyName "registryRead" -KeyType "secondary"
```

Genera un token SAS Hub molto con il criterio registryRead e la chiave secondaria.

### Esempio 3
```powershell
PS C:\> New-AzIotHubSasToken -ResourceGroupName "myresourcegroup" -IotHubName "myiothub" -DeviceId "myDevice1"
```

Genera un token SAS del dispositivo usando i criteri di iothubowner per accedere al registro di sistema {iothub_name}.

### Esempio 4
```powershell
PS C:\> New-AzIotHubSasToken -ResourceGroupName "myresourcegroup" -IotHubName "myiothub" -DeviceId "myDevice1" -ModuleId "myModule1"
```

Genera un token SAS modulo usando i criteri di iothubowner per accedere al registro di sistema {iothub_name}.
