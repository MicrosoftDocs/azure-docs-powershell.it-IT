# Start-AzBatchPoolResize

## Sinossi
Inizia a ridimensionare un pool.

## SINTASSI

```
Start-AzBatchPoolResize [-Id] <String> [-TargetDedicatedComputeNodes <Int32>]
 [-TargetLowPriorityComputeNodes <Int32>] [-ResizeTimeout <TimeSpan>]
 [-ComputeNodeDeallocationOption <ComputeNodeDeallocationOption>] -BatchContext <BatchAccountContext>
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## Descrizione
Il cmdlet **Start-AzBatchPoolResize** avvia un'operazione di ridimensionamento batch di Azure in un pool.

## ESEMPI

### Esempio 1: ridimensionare un pool in 12 nodi
```
PS C:\>Start-AzBatchPoolResize -Id "ContosoPool06" -TargetDedicatedComputeNodes 12 -BatchContext $Context
```

Questo comando avvia un'operazione di ridimensionamento nel pool che contiene l'ID ContosoPool06.
La destinazione per l'operazione è 12 nodi di calcolo dedicati.
Usa il cmdlet Get-AzBatchAccountKey per assegnare un contesto alla variabile $Context.

### Esempio 2: ridimensionare un pool usando un'opzione di deallocazione
```
PS C:\>Get-AzBatchPool -Id "ContosoPool06" -BatchContext $Context | Start-AzBatchPoolResize -TargetDedicatedComputeNodes 5 -ResizeTimeout ([TimeSpan]::FromHours(1)) -ComputeNodeDeallocationOption ([Microsoft.Azure.Batch.Common.ComputeNodeDeallocationOption]::Terminate) -BatchContext $Context
```

Questo cmdlet consente di ridimensionare un pool in cinque nodi di calcolo dedicati.
Il comando ottiene il pool con ID ContosoPool06 usando il cmdlet Get-AzBatchPool.
Il comando passa l'oggetto pool al cmdlet corrente usando l'operatore pipeline.
Il comando avvia un'operazione di ridimensionamento nel pool.
La destinazione è cinque nodi di calcolo dedicati.
Il comando specifica il periodo di timeout di un'ora.
Il comando specifica un'opzione di disallocazione di terminate.

## PARAMETRI

### -BatchContext
Specifica l'istanza di **BatchAccountContext** utilizzata dal cmdlet per interagire con il servizio batch.
Se si usa il cmdlet Get-AzBatchAccount per ottenere il BatchAccountContext, l'autenticazione di Azure Active Directory verrà usata durante l'interazione con il servizio batch. Per usare invece l'autenticazione con chiave condivisa, usa il cmdlet Get-AzBatchAccountKey per ottenere un oggetto BatchAccountContext con i relativi tasti di scelta popolati. Quando si usa l'autenticazione con chiave condivisa, il tasto di scelta principale viene usato per impostazione predefinita. Per cambiare la chiave da usare, imposta la proprietà BatchAccountContext.
