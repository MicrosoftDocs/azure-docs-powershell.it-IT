# New-AzServiceFabricApplicationTypeVersion

## Sinossi
Creare una nuova versione del tipo di applicazione sotto il gruppo e il cluster di risorse specificati.

## SINTASSI

```
New-AzServiceFabricApplicationTypeVersion [-ResourceGroupName] <String> [-ClusterName] <String>
 [-Name] <String> [-Version] <String> -PackageUrl <String> [-DefaultParameter <Hashtable>] [-Force]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Descrizione
Questo cmdlet crea una nuova versione del tipo di applicazione usando il pacchetto specificato in-PackageUrl, che dovrebbe essere accessibile tramite un endpoint REST per Azure Resource Manager da usare durante la distribuzione e conteneva l'applicazione compressa e zippata con l'estensione sfpkg. Questo comando creerà il tipo di applicazione se non esiste già.

## ESEMPI

### Esempio 1
```powershell
PS C:\> $resourceGroupName = "testRG"
PS C:\> $clusterName = "testCluster"
PS C:\> $appTypeName = "testAppType"
PS C:\> $version = "v1"
PS C:\> $packageUrl = "https://sftestapp.blob.core.windows.net/sftestapp/testApp_1.0.sfpkg"
PS C:\> New-AzServiceFabricApplicationTypeVersion -ResourceGroupName $resourceGroupName -ClusterName $clusterName -Name $appTypeName -Version $version -PackageUrl $packageUrl -Verbose
```

Questo esempio creerà una versione di tipo applicazione "V1" in tipo "testAppType". La versione nel manifesto dell'applicazione contenuta nel pacchetto deve avere la stessa versione di quella specificata in versione.
