# New-AzNotificationHubAuthorizationRule

## SYNOPSIS
Crea una regola di autorizzazione e la assegna a un hub di notifica.

## SINTASSI

### InputFileParameterSet
```
New-AzNotificationHubAuthorizationRule [-ResourceGroup] <String> [-Namespace] <String>
 [-NotificationHub] <String> [-InputFile] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### SASRuleParameterSet
```
New-AzNotificationHubAuthorizationRule [-ResourceGroup] <String> [-Namespace] <String>
 [-NotificationHub] <String> [-SASRule] <SharedAccessAuthorizationRuleAttributes>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIZIONE
Il cmdlet **New-AzNotificationHubAuthorizationRule** crea una regola di autorizzazione Firma di accesso condiviso dell'hub di notifica.
Le regole di autorizzazione vengono usate per gestire l'accesso agli hub di notifica.
Questa operazione viene eseguita con la creazione di collegamenti, come URI, in base a diversi livelli di autorizzazione.
I client vengono indirizzati a uno di questi URI in base al livello di autorizzazione appropriato.
Ad esempio, un client con l'autorizzazione di ascolto verrà indirizzato all'URI per tale autorizzazione.

## ESEMPI

### Esempio 1: Creare una regola di autorizzazione dell'hub di notifica
```
PS C:\>New-AzNotificationHubAuthorizationRule -Namespace "ContosoNamespace" -NotificationHub "ContosoInternalHub" -ResourceGroup "ContosoNotificationsGroup" -InputFile "C:\Configuration\ExternalAccessRule.json"
```

Questo comando crea una nuova regola di autorizzazione e la assegna all'hub di notifica denominato ContosoInternalHub.
Questo hub si trova nello spazio dei nomi ContosoCliente e viene assegnato al gruppo di risorse ContosoNotificationsGroup.
Si noti che tutte le informazioni di configurazione per la regola, incluso il nome della regola, verranno prese dal file di input C:\Configuration\ExternalAccessRule.jsattiva.
