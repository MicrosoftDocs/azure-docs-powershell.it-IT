# Restart-AzServiceFabricManagedNodeType

## SYNOPSIS
Riavviare nodi specifici dal tipo di nodo.

## SINTASSI

```
Restart-AzServiceFabricManagedNodeType [-ResourceGroupName] <String> [-ClusterName] <String> [-Name] <String>
 -NodeName <String[]> [-ForceRestart] [-PassThru] [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## DESCRIZIONE
Riavviare nodi specifici dal tipo di nodo. I nodi dell'infrastruttura di servizio verranno disabilitati prima di riavviare le macchine virtuali e averli ri abilitati nuovamente una volta tornati. Se questa operazione viene eseguita nei tipi di nodo primario, l'operazione potrebbe richiedere del tempo perché potrebbe non riavviare tutti i nodi contemporaneamente. Usare -ForceRestart forza l'operazione anche se l'infrastruttura del servizio non è in grado di disabilitare i nodi, ma con cautela, perché ciò potrebbe causare la perdita di dati se nel nodo sono in esecuzione carichi di lavoro con stato.

## ESEMPI

### Esempio 1
```powershell
$rgName = "testRG"
$clusterName = "testCluster"
$NodeTypeName = "nt1"
Restart-AzServiceFabricManagedNodeType -ResourceGroupName $rgName -ClusterName $clusterName  -Name $NodeTypeName -NodeName nt1_0, nt1_3
```

Riavviare il nodo 0 e 3 nel tipo di nodo.
