# Update-AzSentinelIncident

## SYNOPSIS
Aggiornare un evento imprevisto.

## SINTASSI

### IncidentId (impostazione predefinita)
```
Update-AzSentinelIncident -ResourceGroupName <String> -WorkspaceName <String> -IncidentID <String>
 [-Classification <String>] [-ClassificationComment <String>] [-ClassificationReason <String>]
 [-Description <String>]
 [-Label <System.Collections.Generic.IList`1[Microsoft.Azure.Commands.SecurityInsights.Models.Incidents.PSSentinelIncidentLabel]>]
 [-Owner <PSSentinelIncidentOwner>] -Severity <String> -Status <String> -Title <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

```
 [-ClassificationComment <String>] [-ClassificationReason <String>] [-Description <String>]
```

```
 [-ClassificationReason <String>] [-Description <String>]

 [-Owner <PSSentinelIncidentOwner>] -Severity <String> -Status <String> -Title <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIZIONE
Il cmdlet **Update-AzSentinelIncident** aggiorna l'evento imprevisto nell'area di lavoro specificata.
È possibile passare un **oggetto Evento** imprevisto come parametro o usando l'operatore della pipeline oppure in alternativa è possibile specificare i parametri obbligatori.
È possibile usare il *parametro Confirm* $ConfirmPreference Windows PowerShell per controllare se il cmdlet richiede conferma.

## ESEMPI

### Esempio 1
```powershell
PS C:\> Update-AzSentinelIncident -ResourceGroupName "MyResourceGroup" -WorkspaceName "MyWorkspaceName" -IncidentId "MyIncidentId" -Severity High
```

Il comando recupera l'evento imprevisto per *IncidentId* e imposta la *proprietà Severity* su *High.*  Tutte le altre proprietà rimangono invariate.
