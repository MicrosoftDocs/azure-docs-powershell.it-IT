# Reset-AzHubRouter

## SYNOPSIS
Reimposta lo stato RoutingState di una risorsa VirtualHub.

## SINTASSI

### ByVirtualHubName (impostazione predefinita)
```
Reset-AzHubRouter -ResourceGroupName <String> -Name <String> [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByVirtualHubResourceId
```
Reset-AzHubRouter -ResourceId <String> [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByVirtualHubObject
```
Reset-AzHubRouter -InputObject <PSVirtualHub> [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIZIONE
Reimposta lo stato di routing di una risorsa VirtualHub esistente solo se non è stato eseguito il provisioning dello stato di routing dell'hub virtuale.

## ESEMPI

### Esempio 1

```powershell
PS C:\> Reset-AzHubRouter -ResourceGroupName "testRG" -Name "westushub"
```

Reimpostare lo stato di routing dell'hub virtuale usando ResourceGroupName e ResourceName.

### Esempio 2

```powershell
PS C:\> Reset-AzHubRouter -ResourceId "/subscriptions/testSub/resourceGroups/testRG/providers/Microsoft.Network/virtualHubs/westushub"
```

Reimpostare lo stato di routing dell'hub virtuale usando ResourceId.

### Esempio 3

```powershell
PS C:\> Reset-AzHubRouter -InputObject $virtualHub
```

Reimpostare lo stato di routing dell'hub virtuale usando un oggetto di input. L'oggetto di input è di tipo PSVirtualHub.

### Esempio 4

```powershell
PS C:\> Get-AzVirtualHub -ResourceGroupName "testRG" -Name "westushub" | Reset-AzHubRouter
```

Un oggetto hub virtuale esistente può essere recuperato e quindi passato come oggetto di input a Reset-AzHubRouter.
