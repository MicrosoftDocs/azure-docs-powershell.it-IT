# New-AzCustomProvider

## SYNOPSIS
Crea o aggiorna il provider di risorse personalizzato.

## SINTASSI

```
New-AzCustomProvider -Name <String> -ResourceGroupName <String> -Location <String> [-SubscriptionId <String>]
 [-Action <ICustomRpActionRouteDefinition[]>] [-ResourceType <ICustomRpResourceTypeRouteDefinition[]>]
 [-Tag <Hashtable>] [-Validation <ICustomRpValidations[]>] [-DefaultProfile <PSObject>] [-AsJob] [-NoWait]
 [-Confirm] [-WhatIf] [<CommonParameters>]
```

## DESCRIZIONE
Crea o aggiorna il provider di risorse personalizzato.

## ESEMPI

### Esempio 1: Creare un provider personalizzato
```powershell
PS C:\> New-AzCustomProvider -ResourceGroupName myRG -Name Namespace.Type -Location "West US 2" -ResourceType @{Name="CustomRoute1"; Endpoint="https://www.contoso.com/"}


Location  Name             Type
--------  ----             ----
West US 2 Namespace.Type   Microsoft.CustomProviders/resourceproviders
```

Creare un provider di risorse personalizzato

### Esempio 2: Creare un provider personalizzato con associazioni
```powershell
PS C:\> New-AzCustomProvider -ResourceGroupName myRG -Name Namespace2.Type -Location "West US 2" -ResourceType @{Name="CustomRoute1"; Endpoint="https://www.contoso.com/"}, @{Name="Associations"; Endpoint="https://contoso.com/myService", RoutingType="Proxy,Cache,Extension"}

Location  Name             Type
--------  ----             ----
West US 2 Namespace2.Type   Microsoft.CustomProviders/resourceproviders
```

Creare un provider personalizzato con un percorso per le associazioni dei provider personalizzati.

## NOTE

ALIAS

PROPRIETÀ DEI PARAMETRI COMPLESSE

Per creare i parametri descritti di seguito, creare una tabella hash contenente le proprietà appropriate. Per informazioni sulle tabelle hash, eseguire Get-Help about_Hash_Tables.


ACTION <ICustomRpActionRouteDefinition[]>: elenco di azioni che il provider di risorse personalizzato implementa.
  - `Endpoint <String>`: URI endpoint della definizione della route a cui il provider di risorse personalizzato richiederà l'accesso proxy. Può essere in forma di URI flat (ad esempio ' ') o può specificare di instradare tramite un percorso https://testendpoint/ (ad esempio ' https://testendpoint/{requestPath} ')
  - `Name <String>`: nome della definizione di route. Diventa il nome dell'estensione del ARM (ad esempio '/subscriptions/{subscriptionId}/resourceGroups/{resourceGroupName}/providers/Microsoft.CustomProviders/resourceProviders/{resourceProviderName}/{name}')
  - `[RoutingType <ActionRouting?>]`: tipi di routing supportati per le richieste di azione.

RESOURCETYPE <ICustomRpResourceTypeRouteDefinition[]>: elenco di tipi di risorse che il provider di risorse personalizzato implementa.
  - `Endpoint <String>`: URI endpoint della definizione della route a cui il provider di risorse personalizzato richiederà l'accesso proxy. Può essere in forma di URI flat (ad esempio ' ') o può specificare di instradare tramite un percorso https://testendpoint/ (ad esempio ' https://testendpoint/{requestPath} ')
  - `Name <String>`: nome della definizione di route. Diventa il nome dell'estensione del ARM (ad esempio '/subscriptions/{subscriptionId}/resourceGroups/{resourceGroupName}/providers/Microsoft.CustomProviders/resourceProviders/{resourceProviderName}/{name}')
  - `[RoutingType <ResourceTypeRouting?>]`: tipi di routing supportati per le richieste di risorse.

VALIDATION <ICustomRpValids[]>: elenco di convalide da eseguire nelle richieste del provider di risorse personalizzato.
  - `Specification <String>`: collegamento alla specifica di convalida. La specifica deve essere ospitata in raw.githubusercontent.com.
  - `[ValidationType <ValidationType?>]`: tipo di convalida da eseguire rispetto a una richiesta corrispondente.
