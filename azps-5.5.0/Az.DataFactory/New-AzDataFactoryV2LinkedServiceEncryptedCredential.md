# New-AzDataFactoryV2LinkedServiceEncryptedCredential

## SYNOPSIS
Crittografare le credenziali nel servizio collegato con il runtime di integrazione specificato.

## SINTASSI

### ByFactoryName (Default)
```
New-AzDataFactoryV2LinkedServiceEncryptedCredential [-IntegrationRuntimeName] <String>
 [-DefinitionFile] <String> [-Force] [-ResourceGroupName] <String> [-DataFactoryName] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByFactoryObject
```
New-AzDataFactoryV2LinkedServiceEncryptedCredential [-IntegrationRuntimeName] <String>
 [-DefinitionFile] <String> [-Force] [-DataFactory] <PSDataFactory> [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIZIONE
Il New-AzDataFactoryV2LinkedServiceEncryptedCredential cmdlet crittografa le credenziali nel servizio collegato con il runtime di integrazione specificato.

Assicurarsi che siano soddisfatti i prerequisiti seguenti:
* **L'opzione** di accesso remoto Ã¨ abilitata nel runtime di integrazione self-hosted.
* Per eseguire il cmdlet si usa Powershell 7.0 o versione successiva.

## ESEMPI

### Esempio 1: Crittografare le credenziali in un servizio collegato

Questo comando crittografa le credenziali nel file C:\samples\WikiSample\TaxiDemo1.jscon il runtime di integrazione denominato test-selfhost-ir.

```powershell <!-- Aladdin Generated Example --> 
New-AzDataFactoryV2LinkedServiceEncryptedCredential -DataFactoryName WikiADF -DefinitionFile 'C:\samples\WikiSample\TaxiDemo1.json' -IntegrationRuntimeName 'test-selfhost-ir' -ResourceGroupName MyResourceGroup
```
