# Publish-AzureRmApiManagementTenantGitConfiguration

## Sinossi
Pubblica le modifiche da una filiale git al database di configurazione.

[!INCLUDE [migrate-to-az-banner](../../includes/migrate-to-az-banner.md)]

## SINTASSI

```
Publish-AzureRmApiManagementTenantGitConfiguration -Context <PsApiManagementContext> -Branch <String> [-Force]
 [-ValidateOnly] [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## Descrizione
Il cmdlet **Publish-AzureRmApiManagementTenantGitConfiguration** pubblica le modifiche da una filiale git al database di configurazione.
In alternativa, Ã¨ possibile convalidare le modifiche in un ramo git senza pubblicare.

## ESEMPI

### Esempio 1: distribuire le modifiche git
```
PS C:\>$apimContext = New-AzureRmApiManagementContext -ResourceGroupName "Api-Default-WestUS" -ServiceName "contoso"
PS C:\>Publish-AzureRmApiManagementTenantGitConfiguration -Context $apimContext -Branch 'master' -PassThru
```

Questo comando pubblica le modifiche dal ramo specificato al database di configurazione.

### Esempio 2: convalidare le modifiche git
```
PS C:\>$apimContext = New-AzureRmApiManagementContext -ResourceGroupName "Api-Default-WestUS" -ServiceName "contoso"
PS C:\>Publish-AzureRmApiManagementTenantGitConfiguration -Context $apimContext -Branch 'master' -ValidateOnly -PassThru
```

Questo comando Convalida le modifiche nel ramo git rispetto al database di configurazione.
Le modifiche non vengono pubblicate.
