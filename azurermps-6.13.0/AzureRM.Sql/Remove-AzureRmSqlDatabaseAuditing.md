# Remove-AzureRmSqlDatabaseAuditing

## Sinossi
Rimuove il controllo di un database.

[!INCLUDE [migrate-to-az-banner](../../includes/migrate-to-az-banner.md)]

## SINTASSI

```
Remove-AzureRmSqlDatabaseAuditing [-PassThru] [-ServerName] <String> [-DatabaseName] <String>
 [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## Descrizione
Il cmdlet **Remove-AzureRmSqlDatabaseAuditing** rimuove il controllo di un database SQL di Azure.
Per usare questo cmdlet, usare i parametri *ResourceGroupName* , *ServerName* e *DatabaseName* per identificare il database.
Dopo l'esecuzione di questo cmdlet, il controllo del database non viene eseguito.
Se il comando riesce ed è stato usato il parametro *PassThru* , il cmdlet restituisce un oggetto che descrive il criterio di controllo corrente, oltre agli identificatori del database.
Gli identificatori di database includono, ma non sono limitati a, **ResourceGroupName** , **nomeserver** e **DatabaseName**.
Se si rimuove il controllo di un database SQL di Azure, viene rimosso anche il rilevamento delle minacce.
Questo cmdlet è supportato anche dal servizio database stretch di SQL Server in Azure.

## ESEMPI

### Esempio 1: rimuovere il controllo di un database SQL di Azure
```
PS C:\>Remove-AzureRmSqlDatabaseAuditing -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01"
```

Questo comando rimuove il controllo del database denominato Database01.
Tale database si trova in Server01, assegnato al gruppo di risorse denominato ResourceGroup01.
