# Use-AzureRmSqlServerAuditingPolicy

## Sinossi
Specifica che un database usa i criteri di controllo del server host.

[!INCLUDE [migrate-to-az-banner](../../includes/migrate-to-az-banner.md)]

## SINTASSI

```
Use-AzureRmSqlServerAuditingPolicy [-PassThru] [-ServerName] <String> [-DatabaseName] <String>
 [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## Descrizione
Il cmdlet **use-AzureRmSqlServerAuditingPolicy** specifica che un database usa i criteri di controllo del server host.
Specificare i parametri *ResourceGroupName* , *ServerName* e *DatabaseName* per identificare il database.
Se non è stato definito alcun criterio di controllo per il server di database, il cmdlet non riesce.
Se l'host usa il controllo a livello di server, il rilevamento delle minacce viene rimosso.

## ESEMPI

### Esempio 1: configurare un database per l'uso dei criteri di controllo del server
```
PS C:\>Use-AzureRmSqlServerAuditingPolicy -ResourceGroupName "ResourceGroup01" -ServerName "Server02" -DatabaseName "Database01"
```

Questo comando specifica che il database denominato Database01 in Server02 usa i criteri di controllo del server.
