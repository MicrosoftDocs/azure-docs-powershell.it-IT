# Set-AzureRmSqlServerAuditing

## Sinossi
Modifica le impostazioni di controllo di un server SQL di Azure.

[!INCLUDE [migrate-to-az-banner](../../includes/migrate-to-az-banner.md)]

## SINTASSI

```
Set-AzureRmSqlServerAuditing -State <String> [-AuditActionGroup <AuditActionGroups[]>] [-PassThru]
 [-StorageAccountName <String>] [-StorageKeyType <String>] [-RetentionInDays <UInt32>] [-ServerName] <String>
 [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## Descrizione
Il cmdlet **set-AzureRmSqlServerAuditing** modifica le impostazioni di controllo di un server SQL di Azure.
Per usare il cmdlet, usare i parametri *ResourceGroupName* e *ServerName* per identificare il server.
Specifica il parametro *StorageAccountName* per specificare l'account di archiviazione per i registri di controllo e il parametro *StorageKeyType* per definire le chiavi di archiviazione.
Usa il parametro *state* per abilitare o disabilitare il criterio.

È anche possibile definire la conservazione per i registri di controllo impostando il valore del parametro *RetentionInDays* per definire il periodo per i registri di controllo.

Dopo l'esecuzione del cmdlet, è abilitato il controllo dei database SQL di Azure definiti nell'Azure SQL Server specificato.
Se il cmdlet riesce e si usa il parametro *PassThru* , viene restituito un oggetto che descrive i criteri di controllo BLOB correnti oltre agli identificatori del server.
Gli identificatori del server includono, ma non sono limitati a, **ResourceGroupName** e **ServerName**.

## ESEMPI

### Esempio 1: abilitare i criteri di controllo di un server SQL di Azure
```
PS C:\>Set-AzureRmSqlServerAuditing -State Enabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -StorageAccountName "Storage22"
```

### Esempio 2: disabilitare i criteri di controllo di un server SQL di Azure
```
PS C:\>Set-AzureRmSqlServerAuditing -State Disabled -ResourceGroupName "ResourceGroup01" -ServerName "Server01"
```
