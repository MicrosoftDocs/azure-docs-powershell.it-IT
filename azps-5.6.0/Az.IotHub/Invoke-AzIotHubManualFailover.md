# Invoke-AzIotHubManualFailover

## SYNOPSIS
Richiamare il processo di failover per l'hub IoT nell'area di ripristino di emergenza con associazione geografica.

## SINTASSI

### ResourceSet (impostazione predefinita)
```
Invoke-AzIotHubManualFailover [-ResourceGroupName] <String> [-Name] <String> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObjectSet
```
Invoke-AzIotHubManualFailover [-InputObject] <PSIotHub> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdSet
```
Invoke-AzIotHubManualFailover [-ResourceId] <String> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIZIONE
Attiverà il failover dell'hub IoT nella posizione secondaria. Questa azione causerà tempi di insod>e perdite di telemetria nella soluzione. Si tratta di un'operazione a esecuzione lunga che può richiedere alcuni minuti. Esercitarsi con cautela quando lo si usa.

## ESEMPI

### Esempio 1
```powershell
PS C:\> Invoke-AzIotHubManualFailover -ResourceGroupName "myresourcegroup" -Name "myiothub"
```

Avvio del processo di failover dell'hub IoT "myiothub".

### Esempio 2
```powershell
PS C:\> Invoke-AzIotHubManualFailover -ResourceGroupName "myresourcegroup" -Name "myiothub" -AsJob
```

Avvio del processo di failover dell'hub IoT "myiothub" in background.
