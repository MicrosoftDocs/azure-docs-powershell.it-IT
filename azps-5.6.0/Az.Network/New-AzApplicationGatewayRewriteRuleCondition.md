# New-AzApplicationGatewayRewriteRuleCondition

## SYNOPSIS
Aggiunge una condizione a RewriteRule per un gateway applicazione.

## SINTASSI

```
New-AzApplicationGatewayRewriteRuleCondition -Variable <String> [-Pattern <String>] [-IgnoreCase] [-Negate]
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## DESCRIZIONE
Il cmdlet **AzApplicationGatewayRewriteRuleCondition crea** una condizione della regola di riscrittura per un gateway applicazione di Azure.

## ESEMPI

### Esempio 1
```powershell
PS C:\> $condition = New-AzApplicationGatewayRewriteRuleCondition -Variable "var_request_uri" -Pattern "http" -IgnoreCase
PS C:\> $condition

Variable   : var_request_uri
Pattern    : http
IgnoreCase : True
Negate     : False

PS C:\> $condition | Format-Table

Variable        Pattern IgnoreCase Negate
--------        ------- ---------- ------
var_request_uri http          True  False
```
Questo comando crea una condizione in una regola di riscrittura e archivia il risultato nella variabile $condition.
