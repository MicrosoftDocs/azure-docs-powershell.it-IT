# Set-AzIoTDeviceProvisioningServiceCertificate

## SYNOPSIS
Verificare un certificato del servizio di provisioning dei dispositivi Hub IoT di Azure.

## SINTASSI

### ResourceSet (impostazione predefinita)
```
Set-AzIoTDeviceProvisioningServiceCertificate [-ResourceGroupName] <String> [-Name] <String>
 [-CertificateName] <String> [-Etag] <String> [-Path] <String> [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObjectSet
```
Set-AzIoTDeviceProvisioningServiceCertificate [-InputObject] <PSCertificateResponse> [-Path] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdSet
```
Set-AzIoTDeviceProvisioningServiceCertificate [-ResourceId] <String> [-Etag] <String> [-Path] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIZIONE
Verificare un certificato caricando un certificato di verifica contenente il codice di verifica ottenuto chiamando generate-verification-code. Questo è l'ultimo passaggio della prova del processo di approvazione.
Per una spiegazione dettagliata dei certificati CA nel servizio di provisioning dei dispositivi Hub IoT di Azure, vedi https://docs.microsoft.com/azure/iot-dps/how-to-verify-certificates

## ESEMPI

### Esempio 1
```
PS C:\> Set-AzIoTDeviceProvisioningServiceCertificate -ResourceGroupName "myresourcegroup" -Name "myiotdps" -CertificateName "mycertificate" -Path "c:\mycertificate.cer" -Etag "AAAAAAFpGcA="

Subject             : CN=mycertificate
Thumbprint          : 38303FC7371EC78DDE3E18D712C8414EE50969C7
Status              : Verified
Expiry              : 1/01/2027 16:01
Created             : 1/01/2017 16:01
Etag                : AAAAAAFpGcA=
```

Verificare la proprietà della chiave privata "mycertificate".
