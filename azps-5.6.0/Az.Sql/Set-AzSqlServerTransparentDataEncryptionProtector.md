# Set-AzSqlServerTransparentDataEncryptionProtector

## SYNOPSIS
Imposta la protezione di TDE (Transparent Data Encryption) per un SQL server.

## SINTASSI

```
Set-AzSqlServerTransparentDataEncryptionProtector [-Type] <EncryptionProtectorType> [[-KeyId] <String>]
 [-Force] [-AsJob] [-ServerName] <String> [-ResourceGroupName] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIZIONE
Il Set-AzSqlServerTransparentDataEncryptionProtector cmdlet imposta la protezione TDE per un SQL server.
Modificando il tipo di protezione TDE, la protezione verrÃ  ruotata.

## ESEMPI

### Esempio 1: Impostare il tipo di protezione TDE (Transparent Data Encryption) su ServiceManaged
```
PS C:\> Set-AzSqlServerTransparentDataEncryptionProtector -Type ServiceManaged -ServerName 'ContosoServer' -ResourceGroup 'ContosoResourceGroup'
```

Questo comando aggiorna il tipo di protezione TDE di un server a Service Managed.
ResourceGroupName ServerName Type ServerKeyVaultKeyName
----------------- ----------                   ---- ---------------------
ContosoResourceGroup ContosoServer ServiceManaged ServiceManaged

### Esempio 2: Impostare il tipo di protezione Crittografia dati trasparente su Vault chiave di Azure
```
PS C:\> Set-AzSqlServerTransparentDataEncryptionProtector -Type AzureKeyVault -KeyId 'https://contoso.vault.azure.net/keys/contosokey/01234567890123456789012345678901' -ServerName 'ContosoServer' -ResourceGroup 'ContosoResourceGroup'
```

Questo comando aggiorna un server in modo da usare la chiave del Vault della chiave del server con ID https://contoso.vault.azure.net/keys/contosokey/01234567890123456789012345678901 ' ' come protezione TDE.
ResourceGroupName ServerName Type ServerKeyVaultKeyName
----------------- ----------                   ---- ---------------------
ContosoResourceGroup ContosoServer AzureKeyVault contoso_contosokey_01234567890123456789012345678901
