# Get-AzSqlCapability

## SYNOPSIS
Recupera SQL funzionalità di database per l'abbonamento corrente.

## SINTASSI

### FilterResults (Default)
```
Get-AzSqlCapability [-LocationName] <String> [-ServerVersionName <String>] [-EditionName <String>]
 [-ServiceObjectiveName <String>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### DefaultResults
```
Get-AzSqlCapability [-LocationName] <String> [-Defaults] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## DESCRIZIONE
Il cmdlet **Get-AzSqlCapability** ottiene le funzionalità di Azure SQL Database sono disponibili nella sottoscrizione corrente per un'area geografica.
Se si specificano i *parametri ServerVersionName,* *EditionName* o *ServiceObjectiveName,* questo cmdlet restituisce i valori specificati e i predecessori.

## ESEMPI

### Esempio 1: Ottenere funzionalità per l'abbonamento corrente per un'area geografica
```
PS C:\>Get-AzSqlCapability -LocationName "Central US"
Location                : Central US
Status                  : Available
SupportedServerVersions : {12.0, 2.0}
```

Questo comando restituisce le funzionalità per SQL istanze di database nella sottoscrizione corrente per l'area geografica Stati Uniti centrali.

### Esempio 2: Ottenere funzionalità predefinite per l'abbonamento corrente per un'area geografica
```
PS C:\>Get-AzSqlCapability -LocationName "Central US" -Defaults
Location        : Central US
Status          : Available
ExpandedDetails : Version: 2.0 (Default) -> Edition: Standard (Default) -> Service Objective: S0 (Default)
```

Questo comando restituisce le funzionalità predefinite per SQL database della sottoscrizione corrente nell'area centrale degli Stati Uniti.

### Esempio 3: Ottenere i dettagli per un obiettivo del servizio
```
PS C:\>Get-AzSqlCapability -LocationName "Central US" -ServiceObjectiveName "S1"
Location        : Central US
Status          : Available
ExpandedDetails : Version: 12.0 (Available) -> Edition: Standard (Default) -> Service Objective: S1 (Available) 
                  Version: 2.0 (Default) -> Edition: Standard (Default) -> Service Objective: S1 (Available)
```

Questo comando recupera le funzionalità predefinite per SQL database per l'obiettivo di servizio specificato nella sottoscrizione corrente.
