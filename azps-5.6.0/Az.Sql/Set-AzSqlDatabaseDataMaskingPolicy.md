# Set-AzSqlDatabaseDataMaskingPolicy

## SYNOPSIS
Imposta la maschera dati per un database.

## SINTASSI

```
Set-AzSqlDatabaseDataMaskingPolicy [-PassThru] [-PrivilegedUsers <String>] [-DataMaskingState <String>]
 [-ServerName] <String> [-DatabaseName] <String> [-ResourceGroupName] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIZIONE
Il cmdlet **Set-AzSqlDatabaseDataMaskingPolicy** imposta i criteri di maschera dati per un database SQL Azure.
Per usare questo cmdlet, usare i *parametri ResourceGroupName,* *ServerName* e *DatabaseName* per identificare il database.
È possibile impostare il *parametro DataMaskingState* per specificare se le operazioni di maschera dati devono essere abilitate o disabilitate.
Se il cmdlet ha esito positivo e viene usato il *parametro PassThru,* restituisce un oggetto che descrive i criteri di maschera dati correnti oltre agli identificatori del database.
Gli identificatori di database includono, ma non limitati a, **ResourceGroupName,** **ServerName** e **DatabaseName.**
Questo cmdlet è supportato anche dal servizio SQL Server stretch database in Azure.

## ESEMPI

### Esempio 1: Impostare i criteri di maschera dati per un database
```
PS C:\>Set-AzSqlDatabaseDataMaskingPolicy -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01 -PrivilegedUsers "public" -DataMaskingState "Enabled"
```

Questo comando imposta i criteri di maschera dati per un database denominato database01 nel server denominato server01.
