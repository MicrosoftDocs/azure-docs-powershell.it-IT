## DESCRIZIONE
Creare una nuova sottoscrizione di evento per un argomento griglia eventi di Azure, una risorsa Azure supportata, una sottoscrizione di Azure o un gruppo di risorse.
Per creare una sottoscrizione di evento per la sottoscrizione di Azure attualmente selezionata, specificare il nome della sottoscrizione dell'evento e l'endpoint di destinazione.
Per creare una sottoscrizione di un evento a un gruppo di risorse, specificare il nome del gruppo di risorse oltre al nome della sottoscrizione dell'evento e all'endpoint di destinazione.
Per creare una sottoscrizione di un evento per un argomento griglia eventi di Azure, specificare anche il nome dell'argomento.
Per creare una sottoscrizione evento a una risorsa Azure supportata, specificare l'ID risorsa completo della risorsa. Per visualizzare l'elenco dei tipi supportati, eseguire il cmdlet Get-AzEventGridTopicType.

```powershell
PS C:\> New-AzEventGridSubscription -ResourceGroup MyResourceGroup -TopicName Topic1 -Endpoint https://requestb.in/19qlscd1 -EventSubscriptionName EventSubscription1
```

Crea una nuova sottoscrizione evento EventSubscription1 in un argomento Argomento1 della griglia eventi di Azure nel gruppo di risorse \` \` \` \` \` MyResourceGroupName \` con l'endpoint di destinazione webhook. `https://requestb.in/19qlscd1` La sottoscrizione di questo evento usa filtri predefiniti.

### Esempio 2
```powershell
PS C:\> New-AzEventGridSubscription -ResourceGroup MyResourceGroupName -Endpoint https://requestb.in/19qlscd1 -EventSubscriptionName EventSubscription1
```

Crea una nuova sottoscrizione evento \` EventSubscription1 a un gruppo \` di risorse \` MyResourceGroupName \` con l'endpoint di destinazione webhook. `https://requestb.in/19qlscd1` La sottoscrizione di questo evento usa filtri predefiniti.

### Esempio 3
```powershell
PS C:\> New-AzEventGridSubscription -Endpoint https://requestb.in/19qlscd1 -EventSubscriptionName EventSubscription1
```

Crea una nuova sottoscrizione evento EventSubscription1 per la sottoscrizione di Azure attualmente selezionata \` \` con l'endpoint di destinazione `https://requestb.in/19qlscd1` webhook. La sottoscrizione di questo evento usa filtri predefiniti.

### Esempio 4
```powershell
PS C:\> $includedEventTypes = "Microsoft.Resources.ResourceWriteFailure", "Microsoft.Resources.ResourceWriteSuccess"
PS C:\> $labels = "Finance", "HR"
PS C:\> New-AzEventGridSubscription -Endpoint https://requestb.in/19qlscd1 -EventSubscriptionName EventSubscription1 -SubjectBeginsWith "TestPrefix" -SubjectEndsWith "TestSuffix" -IncludedEventType $includedEventTypes -Label $labels
```

Crea una nuova sottoscrizione evento EventSubscription1 per la sottoscrizione di Azure attualmente selezionata \` \` con l'endpoint di destinazione `https://requestb.in/19qlscd1` webhook. Questa sottoscrizione dell'evento specifica i filtri aggiuntivi per i tipi di evento e l'oggetto e solo gli eventi corrispondenti a tali filtri verranno recapitati all'endpoint di destinazione.

### Esempio 5

### Esempio 6

Crea una nuova sottoscrizione evento \` EventSubscription1 in \` uno spazio dei nomi EventHub con l'endpoint di destinazione webhook `https://requestb.in/19qlscd1` specificato. La sottoscrizione di questo evento usa filtri predefiniti.

## PARAMETERS

### -AdvancedFilter
Filtro avanzato che specifica una matrice di più valori Hashtable usati per il filtro basato su attributi. Ogni valore Hashtable contiene le informazioni chiave-valore seguenti: Operazione, Chiave, Valore o Valori. L'operatore può essere uno dei valori seguenti: NumberIn, NumberNotIn, NumberLessWith, NumberGreaterWith, NumberLessThanOrEquals, NumberGreaterThanOrEquals, BoolEquals, StringIn, StringNotIn, StringBeginsWith, StringEndsWith o StringContains. La chiave rappresenta la proprietà payload in cui vengono applicati i criteri di filtro avanzati. Infine, Valore o Valori rappresenta il valore o il set di valori per cui trovare una corrispondenza. Può trattarsi di un singolo valore del tipo corrispondente o di una matrice di valori. Come esempio dei parametri di filtro avanzati: $AdvancedFilters=@($AdvFilter 1; $AdvFilter 2) dove $AdvFilter 1=@{operator="NumberIn"; key="Data.Key1"; Values=@(1,2)} e $AdvFilter 2=@{operator="StringBringsWith"; key="Subject"; Values=@("SubjectPrefix1","SubjectPrefix2")}
