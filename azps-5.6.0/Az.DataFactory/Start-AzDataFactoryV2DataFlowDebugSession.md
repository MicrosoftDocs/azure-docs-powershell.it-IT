# Start-AzDataFactoryV2DataFlowDebugSession

## SYNOPSIS
Avvia una sessione di debug del flusso di dati in Data factory di Azure

## SINTASSI

### ByFactoryName (Default)
```
Start-AzDataFactoryV2DataFlowDebugSession [[-IntegrationRuntimeFile] <String>] [-AsJob]
 [-ResourceGroupName] <String> [-DataFactoryName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByFactoryObject
```
Start-AzDataFactoryV2DataFlowDebugSession [[-IntegrationRuntimeFile] <String>] [-AsJob]
 [-DataFactory] <PSDataFactory> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### ByResourceId
```
Start-AzDataFactoryV2DataFlowDebugSession [[-IntegrationRuntimeFile] <String>] [-AsJob] [-ResourceId] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIZIONE
Questo comando a esecuzione lunga avvia una sessione di debug del flusso di dati per i futuri comandi di debug. Questo comando puÃ² essere associato a una definizione di runtime di integrazione per configurare le dimensioni/tipo del cluster di sessione di debug.
La sequenza di comandi di PowerShell per il flusso di lavoro di debug del flusso di dati deve essere:
1. Start-AzDataFactoryV2DataFlowDebugSession
1. Add-AzDataFactoryV2DataFlowDebugSessionPackage
1. Invoke-AzDataFactoryV2DataFlowDebugSessionCommand (ripetere questo passaggio per i diversi comandi/destinazioni oppure ripetere i passaggi da 2 a 3 per modificare il file del pacchetto)
1. Stop-AzDataFactoryV2DataFlowDebugSession
