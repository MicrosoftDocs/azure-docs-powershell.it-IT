# Set-AzDataFactoryV2Pipeline

## SYNOPSIS
Crea una pipeline in Data factory.

## SINTASSI

### ByFactoryName (Default)
```
Set-AzDataFactoryV2Pipeline [-Name] <String> [-DefinitionFile] <String> [-ResourceGroupName] <String>
 [-DataFactoryName] <String> [-Force] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### ByResourceId
```
Set-AzDataFactoryV2Pipeline [-DefinitionFile] <String> [-ResourceId] <String> [-Force]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIZIONE
Il cmdlet Set-AzDataFactoryV2Pipeline crea una pipeline in Data factory di Azure.
Se si specifica un nome per una pipeline già esistente, il cmdlet richiede conferma prima che sostituisca la pipeline.
Se si specifica il parametro Force, il cmdlet sostituisce la pipeline esistente senza conferma.
Eseguire queste operazioni nell'ordine seguente: -- Creare un data factory.
-- Creare servizi collegati.
-- Creare set di dati.
-- Creare una pipeline.
Se esiste già una pipeline con lo stesso nome nel data factory, questo cmdlet chiede di confermare se sovrascrivere la pipeline esistente con la nuova pipeline.
Se si conferma di sovrascrivere la pipeline esistente, viene sostituita anche la definizione della pipeline.

## ESEMPI

### Esempio 1: Creare una pipeline
```
PS C:\> Set-AzDataFactoryV2Pipeline -ResourceGroupName "ADF" -Name "DPWikisample" -DataFactoryName "WikiADF" -File "C:\DPWikisample.json"

    PipelineName      : DPWikisample
    ResourceGroupName : ADF
    DataFactoryName   : WikiADF11
    Activities        : {MyCopyActivity_0_0, MyCopyActivity_1_0}
    Parameters        : {[OutputBlobName, Microsoft.Azure.Management.DataFactory.Models.ParameterSpecification]}
```

Questo comando crea una pipeline denominata DPWikisample nel data factory denominato ADF.
Il comando basa la pipeline sulle informazioni del DPWikisample.jssu file.
Questo file include informazioni su attività come l'attività di copia e l'attività HDInsight nella pipeline.
