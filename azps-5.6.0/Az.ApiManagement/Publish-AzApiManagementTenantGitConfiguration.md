# Publish-AzApiManagementTenantGitConfiguration

## SYNOPSIS
Pubblica le modifiche da un branch Git nel database di configurazione.

## SINTASSI

```
Publish-AzApiManagementTenantGitConfiguration -Context <PsApiManagementContext> -Branch <String> [-Force]
 [-ValidateOnly] [-PassThru] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## DESCRIZIONE
Il cmdlet **Publish-AzApiManagementTenantGitConfiguration** pubblica le modifiche da un branch Git nel database di configurazione.
In alternativa, Ã¨ possibile convalidare le modifiche in un ramo Git senza pubblicare.

## ESEMPI

### Esempio 1: Distribuire le modifiche di Git
```
PS C:\>$apimContext = New-AzApiManagementContext -ResourceGroupName "Api-Default-WestUS" -ServiceName "contoso"
PS C:\>Publish-AzApiManagementTenantGitConfiguration -Context $apimContext -Branch 'master' -PassThru
```

Questo comando pubblica le modifiche dal ramo specificato nel database di configurazione.

### Esempio 2: Convalidare le modifiche di Git
```
PS C:\>$apimContext = New-AzApiManagementContext -ResourceGroupName "Api-Default-WestUS" -ServiceName "contoso"
PS C:\>Publish-AzApiManagementTenantGitConfiguration -Context $apimContext -Branch 'master' -ValidateOnly -PassThru
```

Questo comando convalida le modifiche nel ramo Git rispetto al database di configurazione.
Le modifiche non vengono pubblicate.
