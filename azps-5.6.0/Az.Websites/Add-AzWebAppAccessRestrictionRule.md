# Add-AzWebAppAccessRestrictionRule

## SYNOPSIS
Aggiunge una regola di ritienizione di Access a un'app Web di Azure.

## SINTASSI

### IpAddressParameterSet (impostazione predefinita)
```
Add-AzWebAppAccessRestrictionRule [-ResourceGroupName] <String> [-WebAppName] <String> [-Name <String>]
 [-Description <String>] -Priority <UInt32> [-Action <String>] [-SlotName <String>] [-TargetScmSite]
 -IpAddress <String> [-PassThru] [-HttpHeader <Hashtable>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## DESCRIZIONE
Il cmdlet **Add-AzWebAppAccessRestrictionRule** aggiunge una regola di restrizione di accesso a un'app Web di Azure.

## ESEMPI

### Esempio 1: Aggiungere una regola di restrizione di accesso IpAddress a un'app Web
```
PS C:\>Add-AzWebAppAccessRestrictionRule -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-Name IpRule -Priority 200 -Action Allow -IpAddress 10.10.0.0/8
```

Questo comando aggiunge una regola di restrizione di accesso con priorità 200 e un intervallo IP a un'app Web denominata ContosoSite che appartiene al gruppo di risorse Default-Web-WestUS.

### Esempio 2: Aggiungere la regola di restrizione di accesso all'endpoint del servizio subnet a un'app Web
```
PS C:\>Add-AzWebAppAccessRestrictionRule -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-Name SubnetRule -Priority 300 -Action Allow -SubnetName appgw-subnet -VirtualNetworkName corp-vnet
```

Questo comando aggiunge una regola di restrizione di accesso con priorità 300 e con subnet appgw-subnet in corp-vnet a un'app Web denominata ContosoSite che appartiene al gruppo di risorse Default-Web-WestUS.

### Esempio 3: Aggiungere una regola di restrizione di accesso ServiceTag a un'app Web
```
PS C:\>Add-AzWebAppAccessRestrictionRule -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-Name ServiceTagRule -Priority 200 -Action Allow -ServiceTag AzureFrontDoor.Backend
```

Questo comando aggiunge una regola di restrizione di accesso con priorità 200 e un tag di servizio che rappresenta l'ambito IP di Azure Front Door a un'app Web denominata ContosoSite che appartiene al gruppo di risorse Default-Web-WestUS.

### Esempio 4: Aggiungere una regola di restrizione di accesso a più indirizzi a un'app Web
```
PS C:\>Add-AzWebAppAccessRestrictionRule -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-Name MultipleIpRule -Priority 200 -Action Allow -IpAddress "10.10.0.0/8,192.168.0.0/16"
```

Questo comando aggiunge una regola di restrizione di accesso con priorità 200 e due intervalli IP a un'app Web denominata ContosoSite che appartiene al gruppo di risorse Default-Web-WestUS.

### Esempio 5: Aggiungere una regola di restrizione di accesso con intestazione http a un'app Web
```
PS C:\>Add-AzWebAppAccessRestrictionRule -ResourceGroupName "Default-Web-WestUS" -WebAppName "ContosoSite" 
-Name MultipleIpRule -Priority 400 -Action Allow -ServiceTag AzureFrontDoor.Backend
-HttpHeader @{'x-forwarded-host' = 'www.contoso.com', 'app.contoso.com'; 'x-azure-fdid' = '355deb06-47c4-4ba4-9641-c7d7a98b913e'}
```

Questo comando aggiunge una regola di restrizione di accesso con priorità 400 per il tag di servizio AzureFrontDoor.Backend e limita ulteriormente l'accesso solo alle intestazioni http di determinati valori a un'app Web denominata ContosoSite che appartiene al gruppo di risorse Default-Web-WestUS.
