# New-AzKeyVaultRoleAssignment

## Sinossi
Assegna il ruolo RBAC specificato all'oggetto Principal specificato, in corrispondenza dell'ambito specificato.

## Descrizione
Usare il `New-AzKeyVaultRoleAssignment` comando per concedere l'accesso.
L'accesso viene concesso assegnando il ruolo RBAC appropriato all'ambito corretto.
L'oggetto dell'assegnazione deve essere specificato.
Per specificare un utente, USA SignInName o i parametri di Azure AD ObjectId.
Per specificare un gruppo di sicurezza, usare il parametro di Azure AD ObjectId.
Per specificare un'applicazione Azure AD, USA i parametri ApplicationId o ObjectId.
Il ruolo che viene assegnato deve essere specificato usando il parametro RoleDefinitionName PR RoleDefinitionId. L'ambito in cui viene concesso l'accesso può essere specificato. Il valore predefinito è l'abbonamento selezionato.

## ESEMPI

### Esempio 1
```powershell
PS C:\> New-AzKeyVaultRoleAssignment -HsmName myHsm -RoleDefinitionName "Managed HSM Policy Administrator" -SignInName user1@microsoft.com

RoleDefinitionName               DisplayName                    ObjectType Scope
------------------               -----------                    ---------- -----
Managed HSM Policy Administrator User 1 (user1@microsoft.com)   User       /
```

Questo esempio assegna il ruolo "amministratore dei criteri HSM gestito" all'utente " user1@microsoft.com " all'ambito superiore.
