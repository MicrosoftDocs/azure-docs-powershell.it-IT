# Remove-AzOperationalInsightsWorkspace

## Sinossi
Rimuove un'area di lavoro.

## SINTASSI

```
Remove-AzOperationalInsightsWorkspace [-ResourceGroupName] <String> [-Name] <String> [-ForceDelete] [-Force]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Descrizione
Il cmdlet **Remove-AzOperationalInsightsWorkspace** Elimina un'area di lavoro esistente.
Se l'area di lavoro è stata collegata a un account esistente tramite il parametro *CustomerID* al momento della creazione, l'account originale non viene eliminato nel portale delle informazioni operative.

## ESEMPI

### Esempio 1: rimuovere un'area di lavoro in base al nome
```
PS C:\>Remove-AzOperationalInsightsWorkspace -ResourceGroupName "ContosResourceGroup" -Name "MyWorkspace"
```

Questo comando rimuove l'area di lavoro denominata Workspace dal gruppo di risorse denominato ContosoResourceGroup.

### Esempio 2: rimuovere un'area di lavoro usando la pipeline e senza conferma
```
PS C:\>Get-AzOperationalInsightsWorkspace -ResourceGroupName "ContosResourceGroup" -Name "MyWorkspace" | Remove-AzOperationalInsightsWorkspace -Force
```

Questo comando usa il cmdlet Get-AzOperationalInsightsWorkspace per ottenere l'area di lavoro denominata Workspace e lo passa al cmdlet **Remove-AzOperationalInsightsWorkspace** usando l'operatore pipeline per rimuoverlo.
Dato che il parametro *Force* è specificato, il comando non richiede prima di rimuovere l'area di lavoro.

### Esempio 3: forza di eliminazione dell'area di lavoro (non può essere recuperato)
```
PS C:\> $workspace = New-AzOperationalInsightsWorkspace -ResourceGroupName $rgname -Name $wsname -Location $wslocation
PS C:\> $workspace | Remove-AzOperationalInsightsWorkspace -ForceDelete
```

Forzare l'eliminazione di un'area di lavoro.
