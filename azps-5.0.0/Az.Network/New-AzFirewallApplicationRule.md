# New-AzFirewallApplicationRule

## Sinossi
Crea una regola di applicazione firewall.

## SINTASSI

### TargetFqdn (impostazione predefinita)
```
New-AzFirewallApplicationRule -Name <String> [-Description <String>] [-SourceAddress <String[]>]
 [-SourceIpGroup <String[]>] -TargetFqdn <String[]> -Protocol <String[]>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### FqdnTag
```
New-AzFirewallApplicationRule -Name <String> [-Description <String>] [-SourceAddress <String[]>]
 [-SourceIpGroup <String[]>] -FqdnTag <String[]> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## Descrizione
Il cmdlet **New-AzFirewallApplicationRule** crea una regola dell'applicazione per Azure firewall.

## ESEMPI

### Esempio 1: creare una regola per consentire tutto il traffico HTTPS da 10.0.0.0
```powershell
New-AzFirewallApplicationRule -Name "https-rule" -Protocol "https:443" -TargetFqdn "*" -SourceAddress "10.0.0.0"
```

Questo esempio crea una regola che consentirà a tutto il traffico HTTPS sulla porta 443 da 10.0.0.0.

### Esempio 2: creare una regola per consentire WindowsUpdate per la subnet 10.0.0.0/24
```powershell
New-AzFirewallApplicationRule -Name "windows-update-rule" -FqdnTag WindowsUpdate -SourceAddress "10.0.0.0/24"
```

Questo esempio crea una regola che consentirà il traffico per gli aggiornamenti di Windows per 10.0.0.0/24 Domain.
