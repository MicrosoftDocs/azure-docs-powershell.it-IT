# Stop-AzureRmDeployment

## Sinossi
Cancal una distribuzione in corso

[!INCLUDE [migrate-to-az-banner](../../includes/migrate-to-az-banner.md)]

## SINTASSI

### StopByDeploymentName (impostazione predefinita)
```
Stop-AzureRmDeployment [-Name] <String> [-PassThru] [-ApiVersion <String>] [-Pre]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### StopByDeploymentId
```
Stop-AzureRmDeployment -Id <String> [-PassThru] [-ApiVersion <String>] [-Pre]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### StopByInputObject
```
Stop-AzureRmDeployment -InputObject <PSDeployment> [-PassThru] [-ApiVersion <String>] [-Pre]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Descrizione
Il cmdlet **Stop-AzureRmDeployment** Annulla una distribuzione all'ambito dell'abbonamento che è stata avviata ma non completata.
Per interrompere una distribuzione, la distribuzione deve avere uno stato di provisioning incompleto, ad esempio il provisioning e non uno stato completato, ad esempio provisioning o failed.

Per creare una distribuzione in ambito abbonamento, usare il cmdlet New-AzureRmDeployment.

Questo cmdlet arresta una sola distribuzione in uso. Usa il parametro *Name* per interrompere una distribuzione specifica.

## ESEMPI

### Esempio 1
```
PS C:\>Stop-AzureRmDeployment -Name "deployment01"
```

Questo comando Annulla una distribuzione in corso "deployment01" nell'ambito corrente della sottoscrizione.

### Esempio 2
```
PS C:\>Get-AzureRmDeployment -Name "deployment01" | Stop-AzureRmDeployment
```

Questo comando ottiene la distribuzione "deployment01" nell'ambito corrente della sottoscrizione e la Annulla.
