## Descrizione
Il cmdlet New-AzureRmADAppCredential può essere usato per aggiungere una nuova credenziale o per eseguire il rollforward delle credenziali per un'applicazione.
L'applicazione viene identificata fornendo l'ID oggetto applicazione o l'ID applicazione.

## ESEMPI

### Esempio 1-creare una nuova credenziale applicativa usando una password

```
PS C:\> $SecureStringPassword = ConvertTo-SecureString -String "password" -AsPlainText -Force
PS C:\> New-AzureRmADAppCredential -ObjectId 1f89cf81-0146-4f4e-beae-2007d0668416 -Password $SecureStringPassword
```

Viene aggiunta una nuova credenziale password al appplication esistente con l'ID oggetto "1f89cf81-0146-4f4e-BEAE-2007d0668416".

### Esempio 2: creare una nuova credenziale applicativa usando un certificato

```
PS C:\> $cer = New-Object System.Security.Cryptography.X509Certificates.X509Certificate 
PS C:\> $cer.Import("C:\myapp.cer") 
PS C:\> $binCert = $cer.GetRawCertData() 
PS C:\> $credValue = [System.Convert]::ToBase64String($binCert)
PS C:\> New-AzureRmADAppCredential -ApplicationId 4589cd6b-3d79-4bb4-93b8-a0b99f3bfc58 -CertValue $credValue -StartDate $cer.GetEffectiveDateString() -EndDate $cer.GetExpirationDateString()
```

Il certificato X509 pubblico con codifica Base64 fornito ("MyApp. cer") viene aggiunto all'applicazione esistente con l'ID applicazione "4589cd6b-3D79-4bb4-93b8-a0b99f3bfc58".

### Esempio 3-creare una nuova credenziale applicativa tramite piping

```
PS C:\> $SecureStringPassword = ConvertTo-SecureString -String "password" -AsPlainText -Force
PS C:\> Get-AzureRmADApplication -ObjectId 1f89cf81-0146-4f4e-beae-2007d0668416 | New-AzureRmADAppCredential -Password $SecureStringPassword
```

Ottiene l'applicazione con l'ID oggetto "1f89cf81-0146-4f4e-BEAE-2007d0668416" e convoglia la New-AzureRmADAppCredential per creare una nuova credenziale dell'applicazione con la password specificata.
